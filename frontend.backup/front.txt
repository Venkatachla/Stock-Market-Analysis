STCOK QUANTITATIVE TRADING DASHBOARD - FRONTEND REQUIREMENTS

================================================================================
PROJECT OVERVIEW
================================================================================

Full-stack trading dashboard for quantitative analysis, stock predictions, 
backtesting, paper trading, and market signal detection. Dark theme modern UI 
with real-time data visualization and API integration.


================================================================================
TECHNOLOGY STACK
================================================================================

Framework: Next.js 14 (App Router)
Language: TypeScript (100% type-safe)
Styling: TailwindCSS 4 + Custom CSS (globals.css)
UI Components: React 18 with Hooks
Charts: Recharts (Line, Area, Bar charts)
Icons: Lucide React
HTTP Client: Axios with timeout/error handling
Package Manager: npm
Build Tool: Turbopack (Next.js built-in)

Node version: 18+
npm version: 9+

Environment Variables:
- NEXT_PUBLIC_API_URL=http://localhost:8000 (Backend API endpoint)

================================================================================
DATA & MOCK DATA STRUCTURE
================================================================================

Backend Fallback System:
- When backend (http://localhost:8000) is unavailable, use comprehensive mock data
- Show blue "Demo Mode" banner on all pages
- Mock data should be realistic and consistent

API Endpoints (with fallback mock data):

1. GET /api/regime
   Returns: { regime: "Bull" | "Bear" | "Neutral", confidence: number }

2. GET /api/predictions
   Returns: Array of predictions with:
   - symbol, price, ml_pred, probability, timestamp

3. GET /api/signals
   Returns: Array of signals with:
   - symbol, type ("BUY" | "SELL"), confidence, price, target, stop_loss

4. GET /api/market-sentiment
   Returns: { sentiment: number (0-100), trend: string }

5. GET /api/prediction/{symbol}
   Returns: Detailed prediction for specific stock

6. GET /api/stock-data/{symbol}
   Returns: Historical price data for charting

7. GET /api/news
   Returns: Array of trading-related news items

8. GET /api/backtest/results
   Returns: Historical backtest performance data

9. GET /api/paper-trading/portfolio
   Returns: Current paper trading portfolio state

10. GET /api/paper-trading/trades
    Returns: List of paper trading transactions


Mock Data Content Required:
- 5+ stock predictions (with various probabilities)
- 10+ trading signals (mixed buy/sell with different confidence)
- 4+ backtest strategies with complete metrics
- 5+ paper trading positions
- Market sentiment score
- 8+ news items
- Historical price data points for charts

================================================================================
ERROR HANDLING & STATE MANAGEMENT
================================================================================

Loading States:
- Page loading: Skeleton screens
- Chart loading: Spinner centered in chart area
- Table loading: Row placeholders
- API timeout: Show error banner + retry button

Error States:
- Network error: Banner with retry (auto-fallback to mock data)
- Invalid input: Form validation messages (inline, below input)
- 404 errors: Redirect to 404 page with home link
- 500 errors: Friendly message, contact support

Empty States:
- No data: "No signals available", centered message
- Empty search: "No results for [query]", suggest alternatives
- Empty portfolio: "No positions", show tutorial

State Management:
- Client state: React hooks (useState, useContext)
- API state: React Query or Axios with caching
- Global state: Context API or Zustand (if needed)
