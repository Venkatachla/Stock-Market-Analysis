# Blockchain + Deep Learning Research Gap Plan (Add-Only)

## Paper Summary (High-Frequency Stock Market Price Prediction Using Blockchain and Deep Learning)
- Uses LSTM-based prediction for return-rate estimation.
- Uses Adam optimizer for model training.
- Mentions OGSO (Opposition Glowworm Swarm Optimization) for tuning learning rate/hyperparameters.
- Evaluates mostly with MSE and MAPE.
- Discusses blockchain mainly as a trust/security layer for immutable transaction/data records.
- Mentions Ethereum return-rate as benchmark context in experiments.

## What You Already Have in This Project
- Ensemble prediction stack: XGBoost + LightGBM + RandomForest + optional LSTM.
- LSTM model already trained with Adam optimizer.
- Multi-timeframe prediction and live inference APIs.
- Signal/risk/entry-stop-target generation.
- Backtest and performance dashboards.
- News sentiment integration and timeframe-aware intraday behavior.

## What Is Missing Compared to the Paper Theme
- No blockchain ledger for immutable storage of predictions/signals/trades.
- No OGSO or other metaheuristic hyperparameter tuner for LSTM.
- No dedicated MAPE/MSE reporting endpoint for model evaluation.
- No on-chain proof/verifiability for generated signals.

## Add-Only Implementation Roadmap (No Rewrite)

### Phase 1: "Blockchain Proof Layer" (Fastest for Faculty Demo)
1. Keep your existing DB and APIs unchanged.
2. Add a blockchain anchor module:
   - Compute SHA-256 hash of each prediction/signal payload.
   - Store hash + timestamp + previous hash in local chain table (`chain_blocks`).
   - Optional: periodically anchor Merkle root on public testnet (Polygon Mumbai).
3. Add endpoints:
   - `POST /chain/anchor-latest`
   - `GET /chain/verify/{prediction_id}`
   - `GET /chain/audit`
4. Add UI badge in signal cards:
   - `Verified on-chain` / `Not anchored yet`.

### Phase 2: "Research Metric Layer"
1. Add evaluation API:
   - `GET /metrics/model-quality?symbol=...&timeframe=...`
2. Return:
   - MSE, MAPE, directional accuracy, hit ratio, calibration error.
3. Show these metrics in Backtest and Performance pages.

### Phase 3: "Advanced Tuning Layer"
1. Add hyperparameter tuning module for LSTM:
   - Start with Optuna (practical replacement for OGSO).
   - Optionally add PSO/Glowworm implementation as research extension.
2. Save best params to model registry table (`model_versions`).
3. Add endpoint:
   - `POST /train/lstm-tune`

## Suggested Faculty Demo Narrative
